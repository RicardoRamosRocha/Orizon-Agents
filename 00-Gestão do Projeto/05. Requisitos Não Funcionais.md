# Requisitos Não Funcionais

## Objetivo

Definir os requisitos mínimos de qualidade, segurança e operação do MVP do Orizon Agents.

## Arquitetura

- Utilizar ASP.NET Core 8.
- Adotar Clean Architecture.
- Manter separação entre Domain, Application, Infrastructure e apresentação.
- Preparar a solução para evolução modular.

## Multi-Tenant

- Toda organização deverá acessar apenas seus próprios dados.
- As entidades principais deverão possuir identificação do tenant.
- O isolamento entre clientes deverá ser validado em todas as operações.

## Segurança

- Utilizar autenticação e autorização.
- Não armazenar senhas, tokens ou chaves em texto simples.
- Manter segredos fora do repositório Git.
- Utilizar HTTPS em produção.
- Registrar ações administrativas importantes.

## Banco de Dados

- Utilizar PostgreSQL.
- Utilizar Entity Framework Core.
- Versionar alterações por migrations.
- Manter ambientes de desenvolvimento e produção separados.

## Desempenho

- Utilizar paginação em listagens.
- Evitar consultas desnecessárias.
- Processar mensagens e tarefas demoradas de forma assíncrona.
- Aplicar cache somente quando necessário.

## WhatsApp

- Validar webhooks recebidos.
- Evitar processamento duplicado de mensagens.
- Registrar mensagens recebidas, enviadas e com falha.
- Respeitar limites da WhatsApp Cloud API.

## Inteligência Artificial

- Permitir abstração de provedores de IA.
- Registrar modelo, consumo de tokens e custo estimado.
- Definir limites de uso por organização.
- Evitar exposição de prompts internos e dados sensíveis.

## Observabilidade

- Gerar logs estruturados.
- Registrar erros e falhas de integração.
- Implementar health checks básicos.
- Permitir rastrear uma mensagem durante todo o processamento.

## Experiência do Usuário

- Interface em português do Brasil.
- Layout responsivo.
- Mensagens de erro claras.
- Padrão visual premium da Orizon Platform.

## Backup e Produção

- Realizar backups automáticos do banco.
- Não publicar segredos no código.
- Utilizar configurações separadas por ambiente.
- Validar build, migrations e variáveis antes do deploy.

## Critérios mínimos do MVP

O MVP estará adequado quando possuir:

- isolamento Multi-Tenant;
- autenticação e permissões;
- integração segura com WhatsApp;
- integração com pelo menos um provedor de IA;
- logs básicos;
- controle de consumo;
- tratamento de erros;
- backup em produção.
