# Requisitos Funcionais

## Objetivo

Este documento descreve as funcionalidades previstas para o MVP do Orizon Agents.

Novos requisitos serão adicionados conforme a evolução da plataforma.

---

# RF-01 – Gestão de Organizações

A plataforma deverá permitir:

- Cadastrar organizações.
- Editar organizações.
- Ativar ou desativar organizações.
- Isolar os dados de cada organização (Multi-Tenant).

---

# RF-02 – Gestão de Usuários

A plataforma deverá permitir:

- Cadastro de usuários.
- Login.
- Recuperação de senha.
- Controle de perfis e permissões.

---

# RF-03 – Gestão de Agentes

A plataforma deverá permitir:

- Criar agentes de IA.
- Editar agentes.
- Ativar ou desativar agentes.
- Configurar personalidade.
- Configurar prompts.
- Testar agentes antes da publicação.

---

# RF-04 – Integração com IA

A plataforma deverá permitir:

- Configurar provedores de IA.
- Selecionar o modelo utilizado pelo agente.
- Configurar parâmetros como temperatura e limite de tokens.
- Registrar consumo de IA.

---

# RF-05 – Integração com WhatsApp

A plataforma deverá permitir:

- Conectar números do WhatsApp.
- Receber mensagens.
- Enviar respostas automáticas.
- Registrar histórico das conversas.

---

# RF-06 – Conversas

O sistema deverá permitir:

- Visualizar conversas.
- Pesquisar conversas.
- Encerrar conversas.
- Registrar mensagens enviadas e recebidas.

---

# RF-07 – Atendimento Humano

O sistema deverá permitir:

- Transferir uma conversa para um atendente.
- Retomar o atendimento pelo agente.
- Registrar quem realizou o atendimento.

---

# RF-08 – Base de Conhecimento

A plataforma deverá permitir:

- Enviar documentos.
- Organizar documentos.
- Associar documentos aos agentes.

---

# RF-09 – Dashboard

A plataforma deverá apresentar indicadores como:

- Conversas.
- Mensagens.
- Agentes ativos.
- Consumo de IA.
- Utilização do plano.

---

# RF-10 – Administração

Administradores deverão poder:

- Gerenciar organizações.
- Gerenciar planos.
- Gerenciar provedores de IA.
- Visualizar métricas da plataforma.

---

# Funcionalidades Futuras

Após a conclusão do MVP, estão previstas funcionalidades como:

- RAG.
- Memória de longo prazo.
- Ferramentas (Tools).
- Fluxos de automação.
- Marketplace.
- White Label.
- API pública.
- Suporte a múltiplos canais além do WhatsApp.

---

# Escopo do MVP

A primeira versão deverá contemplar:

- Multi-Tenant.
- Autenticação.
- Gestão de organizações.
- Gestão de usuários.
- Gestão de agentes.
- Integração com um provedor de IA.
- Integração com WhatsApp Cloud API.
- Histórico de conversas.
- Dashboard básico.
- Controle de planos e limites.
