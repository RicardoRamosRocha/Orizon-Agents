# Roadmap

## Objetivo

Este documento apresenta a evolução planejada do Orizon Agents.

O desenvolvimento será realizado de forma incremental, priorizando a entrega de valor aos clientes e a validação do produto antes da implementação de funcionalidades avançadas.

---

# Sprint 0 — Fundação

Objetivo: preparar a base do projeto.

Entregas:

- Documentação inicial.
- Repositório Git.
- Estrutura da solução.
- Arquitetura.
- Ambiente de desenvolvimento.
- Banco de dados.
- Autenticação.
- Multi-Tenant.

---

# Sprint 1 — MVP

Objetivo: disponibilizar a primeira versão utilizável da plataforma.

Entregas:

- Gestão de organizações.
- Gestão de usuários.
- Gestão de agentes.
- Integração com um provedor de IA.
- Integração com WhatsApp Cloud API.
- Histórico de conversas.
- Dashboard inicial.

---

# Sprint 2 — Evolução

Objetivo: ampliar as funcionalidades do MVP.

Entregas:

- Base de conhecimento.
- Upload de documentos.
- Atendimento humano.
- Ferramentas (Tools).
- Relatórios.
- Melhorias na experiência do usuário.

---

# Sprint 3 — Plataforma

Objetivo: transformar o sistema em uma plataforma completa.

Entregas:

- Memória de longo prazo.
- RAG.
- Fluxos de automação.
- API pública.
- Integrações externas.
- Monitoramento avançado.

---

# Futuro

Após a consolidação do produto, estão previstas funcionalidades como:

- Marketplace de agentes.
- Marketplace de integrações.
- White Label.
- Suporte a múltiplos canais (Telegram, Instagram, Web Chat, Teams, Slack).
- Agentes colaborativos.
- IA multimodal.
- Expansão internacional.

---

# Revisão

O Roadmap deverá ser revisado ao final de cada Sprint, podendo ser ajustado conforme o feedback dos clientes e as prioridades do negócio.
