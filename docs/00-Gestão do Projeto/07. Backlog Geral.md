# Backlog Geral

## Objetivo

Este documento reúne todas as funcionalidades, melhorias e ideias previstas para o Orizon Agents.

O backlog é um documento vivo e será atualizado continuamente durante a evolução do projeto.

---

# Alta Prioridade (MVP)

- Gestão de organizações.
- Gestão de usuários.
- Autenticação.
- Multi-Tenant.
- Gestão de agentes.
- Integração com WhatsApp Cloud API.
- Integração com um provedor de IA.
- Dashboard inicial.
- Histórico de conversas.
- Controle de planos.

---

# Média Prioridade

- Base de conhecimento.
- Upload de documentos.
- Atendimento humano.
- Ferramentas (Tools).
- Relatórios.
- Logs avançados.
- Controle de consumo.
- Configuração de personalidade dos agentes.

---

# Baixa Prioridade

- RAG.
- Memória de longo prazo.
- Marketplace.
- API pública.
- White Label.
- Suporte a múltiplos canais.
- Agentes colaborativos.
- IA multimodal.

---

# Observações

O backlog deverá ser revisado ao final de cada Sprint.

Novas funcionalidades poderão ser adicionadas conforme o feedback dos clientes e a evolução da plataforma.
