# Orizon Agents
# 06. Banco de Dados

---

# Documento

**Projeto:** Orizon Agents

Versão 1.0

---

# 1. Objetivo

O banco de dados será responsável por armazenar todas as informações da plataforma.

Será projetado para:

- Alta performance
- Escalabilidade
- Segurança
- Integridade
- Multi-Tenant

---

# 2. Tecnologia

Banco oficial:

PostgreSQL

Principais vantagens:

- Open Source
- Excelente desempenho
- JSON
- Full Text Search
- Alta confiabilidade
- Escalabilidade

---

# 3. ORM

Será utilizado:

Entity Framework Core

Responsabilidades:

- Migrations
- Mapeamentos
- Consultas
- Persistência

---

# 4. Estrutura Geral

O banco será dividido por módulos.

Exemplo:

Identity

Tenants

Agents

Knowledge

AI

Billing

Audit

Settings

Notifications

---

# 5. Convenções

Todas as tabelas deverão possuir:

Id

CreatedAt

UpdatedAt

CreatedBy

UpdatedBy

TenantId (quando aplicável)

---

# 6. Chaves Primárias

Será utilizado:

GUID

Exemplo:

```
Id UUID
```

---

# 7. Chaves Estrangeiras

Todas as relações utilizarão Foreign Keys.

Exemplo:

```
Conversation

AgentId

↓

Agent
```

---

# 8. Índices

Índices deverão existir para:

- TenantId
- Email
- UserId
- AgentId
- ConversationId
- CreatedAt
- Status

---

# 9. Soft Delete

Campos padrão:

```
IsDeleted

DeletedAt

DeletedBy
```

---

# 10. Auditoria

As alterações importantes poderão gerar registros em:

AuditLogs

Campos:

- Entidade
- Operação
- Usuário
- Tenant
- Data
- Valores

---

# 11. Normalização

Priorizar até 3FN.

Desnormalização apenas quando necessária para performance.

---

# 12. Relacionamentos

Principais agregados:

Tenant

↓

Users

↓

Agents

↓

Knowledge Base

↓

Conversations

↓

Messages

↓

Logs

---

# 13. Performance

Boas práticas:

- índices
- paginação
- consultas assíncronas
- Include apenas quando necessário
- projeções

---

# 14. Versionamento

Toda alteração estrutural será realizada através de:

EF Core Migrations

Nunca alterar o banco manualmente em produção.

---

# 15. Backup

Estratégia:

Backup diário

Backup semanal

Backup mensal

Retenção configurável

---

# 16. Recuperação

Objetivos:

RPO baixo

RTO reduzido

Testes periódicos de restauração

---

# 17. Crescimento

Inicialmente:

```
PostgreSQL Único
```

Futuro:

```
Read Replica

↓

Cluster

↓

Sharding
```

---

# 18. Dados Temporários

Utilizar Redis para:

- cache
- sessões
- rate limit
- filas

Não utilizar PostgreSQL como cache.

---

# 19. Convenções de Nomes

Tabelas:

Plural

Exemplo:

Users

Agents

Conversations

Messages

Colunas:

PascalCase no código

snake_case opcional apenas no banco, caso definido pela equipe.

---

# 20. Estratégia Multi-Tenant

Todas as tabelas de negócio conterão:

TenantId

Filtros globais impedirão acesso entre empresas.

---

# 21. Integridade

Utilizar:

- Foreign Keys
- Constraints
- Unique Index
- Check Constraints

Sempre que possível.

---

# 22. Monitoramento

Monitorar:

- tempo de consulta
- locks
- índices
- crescimento
- conexões
- consumo de disco

---

# Conclusão

O PostgreSQL será a base de persistência da Orizon Agents, oferecendo um ambiente robusto, escalável e preparado para suportar milhares de empresas em uma arquitetura Multi-Tenant. Todas as decisões de modelagem seguirão princípios de integridade, performance e facilidade de evolução, utilizando Entity Framework Core como camada oficial de acesso aos dados.
