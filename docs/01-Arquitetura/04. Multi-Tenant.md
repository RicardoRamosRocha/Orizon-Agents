# Orizon Agents
# 04. Multi-Tenant

---

# Objetivo

A Orizon Agents será uma plataforma SaaS Multi-Tenant.

Uma única infraestrutura atenderá centenas ou milhares de empresas mantendo total isolamento dos dados.

---

# Conceito

Cada empresa é um Tenant.

```
Tenant A

Usuários

Agentes

Conversas

Documentos

Billing

--------------

Tenant B

Usuários

Agentes

Conversas

Documentos

Billing
```

Nenhum Tenant poderá visualizar dados de outro.

---

# Estratégia Adotada

Será utilizado:

## Shared Database

+

## Shared Schema

+

## TenantId

Toda entidade possuirá:

```
TenantId
```

---

# Exemplo

```
Agent

Id

TenantId

Name

Description
```

```
Conversation

Id

TenantId

AgentId

CreatedAt
```

---

# Identificação do Tenant

O Tenant poderá ser identificado através de:

- Login
- Domínio
- Subdomínio
- JWT
- API Key

---

# Fluxo

```
Login

↓

JWT

↓

TenantId

↓

Middleware

↓

Application

↓

Repository

↓

Filtro automático
```

---

# Middleware

Será criado um middleware responsável por:

- identificar o tenant

- validar o tenant

- disponibilizar TenantId

---

# Tenant Context

Será criada uma interface.

```
ITenantContext
```

Responsável por fornecer:

```
TenantId

UserId

Subscription

Plan
```

---

# Filtro Automático

Todos os repositórios deverão aplicar filtro.

Exemplo:

```
WHERE TenantId = CurrentTenant
```

Nunca depender da View ou Controller.

---

# Administração

Existirão dois níveis.

## Plataforma

Administrador da Orizon.

Pode visualizar todos os tenants.

---

## Tenant

Administrador da empresa.

Visualiza apenas seus dados.

---

# Planos

Cada tenant possuirá:

- Plano
- Limites
- Créditos
- Tokens
- Usuários
- Agentes
- Documentos

---

# Isolamento

Toda entidade de negócio possuirá:

```
TenantId
```

Exceções:

- Subscription Plans
- Countries
- Languages
- System Settings

Essas tabelas serão globais.

---

# Segurança

O TenantId nunca será enviado pela interface.

Ele será obtido automaticamente através do usuário autenticado.

---

# Auditoria

Toda operação registrará:

- Tenant

- Usuário

- Data

- Operação

- IP

---

# Escalabilidade

Inicialmente:

```
1 Banco

↓

Todos os Tenants
```

Futuro:

```
Shard 1

Shard 2

Shard 3
```

Ou

```
Banco exclusivo para grandes clientes.
```

A arquitetura já estará preparada.

---

# Backup

Os backups deverão permitir:

- restauração completa

- restauração por tenant

---

# Exclusão

Soft Delete por padrão.

Exclusão definitiva apenas por processos administrativos.

---

# Benefícios

- Economia de infraestrutura

- Facilidade de manutenção

- Atualizações centralizadas

- Alta escalabilidade

- Menor custo operacional

---

# Desafios

- Performance

- Segurança

- Índices eficientes

- Consultas corretas

- Auditoria

Todos esses pontos serão tratados nas demais camadas da arquitetura.

---

# Conclusão

O modelo Multi-Tenant adotado pela Orizon Agents permite atender um grande número de clientes utilizando uma única plataforma, garantindo isolamento, segurança e escalabilidade. Toda a arquitetura, desde o domínio até os repositórios, será construída considerando o Tenant como um conceito central do sistema.
